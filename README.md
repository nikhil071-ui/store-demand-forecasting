# Store Demand Forecasting & Inventory Optimization (End-to-End)

An end-to-end, hiring-ready data analytics project using the **Kaggle "Store Item Demand Forecasting"** dataset (a.k.a. demand-forecasting-kernels-only).  
It demonstrates **EDA → Feature Engineering → ML Forecasting → Inventory KPIs → Dashboard/App**.

## 1) Quickstart
```bash
# 1) Create & activate env (example with pip)
python -m venv .venv
source .venv/bin/activate  # on Windows: .venv\Scripts\activate

# 2) Install deps
pip install -r requirements.txt

# 3) Put Kaggle files
# Download from: https://www.kaggle.com/competitions/demand-forecasting-kernels-only/data
# Place train.csv and test.csv here:
#   data/raw/train.csv
#   data/raw/test.csv

# 4) Run pipeline
python src/etl.py
python src/features.py
python src/train.py
python src/forecast.py

# 5) Launch the Streamlit app
streamlit run app/streamlit_app.py
```

## 3) What This Project Shows Hiring Managers
- Clean, reproducible **data pipeline** (Python + YAML config).
- **Time-aware ML** with lag/rolling features and gradient boosting.
- Business KPIs: **Reorder Level, Safety Stock, Stockout Risk**.
- **Interactive dashboard/app** for decision support.
- **SQL star schema** + sample analytical queries.
- Clear documentation + repo hygiene.

## 4) Notes
- The training script limits to `top_n_series` (by total sales) to keep training fast initially.
  Set to 0 to train **all (store,item)** combinations.
- Forecasts default to **90 days**; change `forecast_horizon_days` in `config.yaml`.
- The Streamlit app reads `artifacts/forecast.csv` and lets you filter by Store/Item.
